# muhammettopal.com

This repository contains the source files for Muhammet Topal's academic portfolio and personal blog.

## Site structure

The site is a static website built from Markdown pages, HTML layouts, CSS, JavaScript, and image or document assets.

Main pages include:

- `index.md`, Home
- `about.md`, About Me
- `research.md`, Research overview
- `portfolio.md`, Portfolio landing page
- `teaching.md`, Teaching overview
- `teaching-experience.md`, Teaching Experience
- `teaching-statement.md`, Teaching Statement
- `teaching-certificate.md`, College Teaching Certificate
- `publications.md`, Publications
- `presentations.md`, Presentations
- `extra-curricular.md`, Extra Curricular landing page
- `extracurricular.md`, Academic Service
- `diversions.md`, Diversions
- `cv.md`, CV
- `contact.md`, Contact
- `blog.md`, My Blog
- `blog-essays.md`, Essays
- `blog-podcasts.md`, Podcasts
- `blog-videos.md`, Videos

Shared styling and behavior are mainly in:

- `assets/css/style.css`
- `assets/js/site.js`
- `_layouts/default.html`
- `_layouts/post.html`

## Updating the site

For small edits, update the relevant Markdown, CSS, or JavaScript files directly on GitHub and commit the changes.

For larger updates, upload the changed files from the latest patch ZIP into the same paths in the repository. If a full site ZIP is used, make sure the repository contents match the ZIP contents and remove old files that are no longer part of the site.

## Contact form

The contact form in `contact.md` uses FormSubmit. The form sends messages to the configured email address and redirects to `/thanks/` after submission.

## Appointment calendar

The Contact page includes a small interactive calendar card. The calendar itself is generated by `assets/js/site.js`. The appointment button links to a Google Calendar booking page. To change the booking link, edit the `href` value of the `Book an appointment` button in `contact.md`.

## Blog posts

Blog posts are stored in `_posts/`. Each post uses front matter for title, date, layout, category, tags, and optional media such as embedded video.

## Deployment notes

The site is intended to be hosted through GitHub Pages or a similar static hosting setup. Keep `CNAME` if the custom domain is still used.
